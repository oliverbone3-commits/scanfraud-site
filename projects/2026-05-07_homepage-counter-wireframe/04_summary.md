## Ключевое решение

**Меняем ровно одну вещь: добавляем FAQ перед CTA.**
Всё остальное в существующей главной — лучше, чем у PM.

## Что отвергаем у PM (с обоснованием в `02_`)

- Лид-форма в герое (5 полей с middle name) — конверсия-убийца, похоже на дефолт Balsamiq
- Две полоски индустрий — mixed taxonomy, дубль `industries.html` хаба
- 3 image+bullets блока — дубли уже существующих feature rows и 2-col cards
- Footer и общие элементы — без изменений (общие для 21 страницы)

## Что берём

- **FAQ-секция** перед финальным CTA. SEO + снятие возражений до запроса демо.

## Что показать PM

`02_structure-proposal.md` — главный документ для презентации. Структура → обоснование → сравнительная таблица → next steps.

`03_faq-content.md` — черновики FAQ. В конце список «что подтвердить» — 8 пунктов: цены, SLA, имя аудитора, формулировки про миграцию.

## Чего НЕ сделано (намеренно)

- **Не правили index.html** — только spec. Если ОК — отдельная задача на верстку.
- **Не делали HTML-мокап** — выбрали markdown по твоему ответу. Если PM не вкатит текст — могу собрать визуальный wireframe второй итерацией.
- **Не правили CONTENT-PLAN.md** — там есть расхождение с live версией (H1 в plan «Unified platform...», на сайте «Pure Traffic. No Friction.»). Это отдельная задача.

## Если предложение одобрено — следующие шаги

1. PM подтверждает контент FAQ + закрывает 8 открытых вопросов
2. Завести задачу: **«Add FAQ section to homepage»**, workflow #6 (New Section)
3. Цепочка агентов: UI Designer → UX Architect → Frontend Developer → Accessibility Auditor → Code Reviewer → Evidence Collector
