### Feature blocks (4)

**01 — Complete data enrichment per event**
Every transaction is automatically enriched with 15+ data points. Even if the merchant sends incomplete data, the system identifies and fills the gaps in real time.
- Identification: BIN, country of origin, issuing bank, and payment system
- Technical profile: IP address, Device ID, and Email fingerprinting
- Financial accuracy: real-time conversion to base currency for precise limit and velocity checks

**02 — Instant root cause analysis**
Eliminate the guesswork behind rejected payments. The monitoring table displays the exact decline reason for every transaction.
- See the specific Rule ID and the condition that triggered the block
- Differentiate between technical failures (provider errors) and risk-based declines

**03 — "For Review" strategic pipeline**
Transactions flagged by specific rules are automatically routed to the For Review queue.
- Analyse suspicious patterns without interrupting the main traffic flow
- One-click actions: add to blacklists or run a Crosscheck analysis to uncover linked fraud networks

**04 — Instant notifications**
Stay informed without refreshing the dashboard.
- Late April: native "In-Platform Bell" notifications for critical events and threshold breaches

## 7. PAYMENT ROUTING & CASCADING (routing-cascading.html)

### Hero
- **Eyebrow:** Products / Payment Routing & Cascading
- **H1:** Route every payment to the right provider.
- **Subtitle:** Optimise payment flows with rule-based routing. Automatic cascading rescues failed transactions. No dev effort required.

### Key features
- **Smart provider selection** — Route to the most cost-effective provider automatically. Save on fees.
- **Automated re-routing** — If a provider is down or approaching thresholds, transactions reroute automatically.
- **Traffic distribution** — Manage and distribute traffic across providers based on your rules.

### Payment Routing section
Routes transactions to the best provider based on predefined or dynamic parameters.

**Static parameters:** Geolocation, Amount, Date & time, Buyer data, Currency, Card issuer/type/brand, Authentication mode, Additional payment details

**Dynamic parameters:** Provider fees, Provider limits & restrictions, Conversion & acceptance rates, Turnovers, Traffic ratios, Balances

### Payment Cascading section
Resubmits a failed transaction to an alternative provider automatically — without degrading the customer experience.

**Sequential parameters:** Provider sequence, Geolocation, Amount, Transaction type, Currency, Card brand

**Adaptive parameters:** Provider performance, Provider availability, Approval ratios, Account balances, Real-time volume, Provider restrictions

### Why choose this
- Intuitive no-code interface
- Quick rule creation and management
- No technical expertise required
- Supports manual and parameter-based priority
- Seamless multi-provider integration
- Reliable fallback options
