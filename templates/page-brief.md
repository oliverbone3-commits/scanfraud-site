# Page Brief Template

**Назначение.** PM (или любой инициатор задачи) заполняет этот шаблон **до** того, как начинать макет/wireframe/верстку. Без заполненного брифа задача не принимается в работу.

**Как использовать.**
1. Скопируй файл в `projects/YYYY-MM-DD_short-name/00_brief.md`
2. Заполни все секции. Если поле неприменимо — пиши «N/A» с обоснованием.
3. Передай ссылку команде/агенту, который будет делать.

## 2. Аудитория / ICP

Кто именно читает страницу. Выбрать одно или несколько.

- [ ] High-risk merchant (gambling, crypto, adult, CBD, forex)
- [ ] Mainstream merchant (e-commerce, fintech)
- [ ] PSP / EMI / acquirer (B2B-инфраструктура)
- [ ] Risk / compliance team (technical evaluator)
- [ ] CFO / commercial decision-maker
- [ ] Другое: ____________

**ICP-уровень внимания.** Critical для копирайта и тона:
- [ ] **Holistic** — все merchants независимо от vertical
- [ ] **High-risk priority** — основное послание для high-risk; mainstream получает сигнал но не первичный
- [ ] **High-risk only** — лобовое позиционирование «for verticals others won't touch»

## 6. Блоки страницы

Выбрать из `system/block-catalog.md`. Указать порядок сверху вниз.

| # | Block (из catalog) | Position | Status | Notes |
|---|--------------------|----------|--------|-------|
| 1 | sticky nav | top | KEEP existing | |
| 2 | hero (split) | above-fold | NEW | Visual: ____ |
| 3 | … | | | |

**Status коды:**
- `KEEP existing` — уже есть на сайте, не трогаем
- `REVISE` — есть, но переделываем (указать что именно)
- `NEW` — новый блок
- `REMOVE` — удаляем (указать причину)
